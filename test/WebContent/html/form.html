입력양식 실습

form 태그 : 사용자의 입력을 받는 화면을 구성할 때 사용하는 태그로 입력한 데이터를 다른 페이지로 보낼 수 있는 태그.
예) 로그인 할 때 아이디와 비밀번호를 입력하여 보내는 경우
예) 회원가입 할 때 가입에 필요한 여러 회원정보를 입력하여 보내는(전송할 수 transfer) 경우 등..

입력(input)태그

text한 줄 택스트를 입력하는 태그
submit폼 태그내의 입력데이터를 전송하는 버튼 태그
password입력 텍스트가 보이지 않는 비밀번호 태그
hidden실제 소스가 존재하지만 화면에서는 보이지 않는 히든 태그
radio여러항목 중 한개만 선택할 때 사용하는 태그
checkbox여러 항목을 모두 선택할 수 있는 태그
file첨부할 파일을 선택하는 태그

form 태그를 통해 입력 데이터를 내보낼 때(전송) 반드시 name속성이 있어야 한다.

form 태그의 action 속성값이 없거나 action을 지정하지 않으면 자기자신페이지를 연다.

입력태그 자동완성기능 : autocomplete로 기본은 on상태이므로, 자동완성기능을 사용 하지 않으려면 off로 바꿔줘야한다.
autocomplete='off'

전송되는 입력값들을 파라미터라고 한다.
전송하는 데이터가 여러개인 경우 각 파라미터는 & 로 구분되어 있다.
파라미터가 노출이 되는 형태, 노출되지않는 형태가 있는데 노출되는 형태는 method:get 방식, 노출되지않는형태 method:post 방식

placeholder: 텍스트 입력태그에 무엇을 입력하는 태그인지 보여지게 하는 속성

required : 반드시 입력해야만 하는 필수입력항목을 입력하지 않으면 submit되지 않도록 처리하려면 프로그래밍적으로 해결할 수도 있지만 태그 자체의 required 속성을 이용할 수도 있다.

readonly : 입력태그이지만 변경 불가능하게 처리하고자 할 때 지정하는 속성(데이터 전송 된다.)
disabled : readonly와 같은 내용이지만 차이점으로는 submit시 데이터 전송이 안된다.

파일을 첨부하여 전송하는 경우는 form에 반드시 지정해야 할 속성 2가지
1. method를 post로 지정해야만 한다.
2. enctype을 multipart/form-data 를 지정해야만 한다.
